# R223R v0.1 vs v0.2 comparison plan

## Purpose

The pilot must not ask "is v0.2 newer?" It must ask whether v0.2 candidate improves teacher-readable generation and review traceability without making teacher drafts more technical.

## Comparison axes

| axis | v0.1 baseline | v0.2 candidate expected improvement | failure signal |
| --- | --- | --- | --- |
| Unit position | often implicit | explicit unit_phase_role and lesson_position_in_unit | all lessons expand alike |
| Practice density | mostly inferred | practice_intensity affects classroom expansion density | intro lesson becomes heavy creation workshop |
| Pattern routing | broad classroom events | practice_pattern_type + conditional fields | pattern names leak into teacher draft |
| Teacher manuscript | mature draft expected | same maturity, better density fit | field wall or ledger language |
| Review ledger | basic classroom event trace | stronger screen/component/sheet/evidence trace | missing component status or evidence outputs |
| Component safety | component hints only | explicit already_registered/candidate/new_surface status | component buttons or execution language |
| Cross-sample separation | sample-specific content | no cross-sample contamination | stationery language appears in paper/color lessons |

## Required sample comparisons

1. M_stationery
   - v0.2 should show heavier creative support and teacher observation points.
   - v0.2 must not turn into component/tool catalog.

2. N_paper_print
   - v0.2 should show material prediction, trial print evidence and technique comparison.
   - v0.2 must not import stationery design or color collision content.

3. O_color_collision
   - v0.2 should keep color perception, experiment and expression as the main classroom line.
   - v0.2 must not become a large making project unless unit role changes.

## Evaluation process

1. Generate or load fixture v0.1 draft.
2. Generate or load fixture v0.2 candidate draft.
3. Compare teacher default view first.
4. Inspect review ledger second.
5. Record density difference and field leakage risk.
6. Stop if teacher default draft becomes less readable than v0.1.
