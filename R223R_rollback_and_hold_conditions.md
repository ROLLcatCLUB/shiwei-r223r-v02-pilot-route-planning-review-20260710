# R223R rollback and hold conditions

## Immediate hold conditions

R223S must be held if any condition occurs:

1. Teacher default draft shows backend field names.
2. Teacher default draft becomes a card wall or component shelf.
3. New surface candidate enters teacher default draft as a tool name.
4. Review ledger loses unit intensity router fields.
5. Review ledger loses component status.
6. Screen / learning sheet / evidence mapping no longer derives from classroom events.
7. v0.2 candidate content contaminates another sample.
8. All lesson types expand at the same density.
9. Pilot implies formal apply or lesson body writeback.
10. Pilot requires R97B route or formal UI changes.

## Rollback rule

Since R223S should be opt-in sandbox / fixture-only, rollback must mean:

- delete sandbox fixture preview files
- keep review package for audit
- do not revert app code because no app code should be modified
- keep R223M v0.1 candidate and P5 v0.2 candidate records unchanged
